## 2. RLS вЂ” FOUND OK (one real caveat)
Per the pulled schema, RLS is **ENABLED** on all three tables, each with `auth.uid()` ownership policies:
- **profiles**: SELECT + UPDATE on `auth.uid() = id`. (INSERT handled by the `handle_new_user` SECURITY DEFINER trigger вЂ” users don't insert directly. No DELETE = denied.)
- **usage**: SELECT + UPDATE + INSERT on `auth.uid() = user_id`.
- **worksheets**: SELECT + INSERT (WITH CHECK) + UPDATE on `auth.uid() = user_id`. No DELETE = denied.

Notes:
- UPDATE policies omit an explicit `WITH CHECK`, but Postgres falls back to the `USING` expression for the check on UPDATE вЂ” so they're **not** wide open. OK.
- `GRANT ALL ... TO anon/authenticated` is standard Supabase; RLS is the real gate (anon has no `auth.uid()` в†’ sees no rows). OK.
- вљ пёЏ **CAVEAT (ties into #6):** the `usage` UPDATE/INSERT policies let a user write **their own** usage row. That means a user can directly reset `worksheets_used` to 0 and **bypass the free-tier limit**. The limit is only truly enforceable if usage writes are **service-role only**. Flagged; real fix is a design change (see #6).
- рџ”Ћ **Live verification still recommended:** run
  `SELECT tablename, rowsecurity FROM pg_tables WHERE schemaname='public';`
  and `SELECT * FROM pg_policies WHERE schemaname='public';`
  to confirm the live DB matches this pulled schema.

## 5. Edge function auth + ownership вЂ” FOUND ISSUE
`supabase/functions/fill-worksheet/index.ts`:
- вќЊ Does **not** read/verify the `Authorization` JWT.
- вќЊ Does **not** check that `worksheetId`/`storagePath` belongs to the caller.
- It uses the **service role** to download `storagePath` and process it. So any caller holding the public anon key (it's in the app bundle) can invoke with an arbitrary `worksheetId`/`storagePath` and have the function read/process files вЂ” and run up the Anthropic bill. (`verify_jwt` defaults to true, but the anon key is itself a valid JWT, so that alone is not user-level protection.)
- **Planned fix (item 5):** require `Authorization`, `getUser(token)`, then fetch the worksheet row and require `worksheet.user_id === user.id` (401/403 otherwise). Also update `lib/worksheet.ts fillWorksheet()` to forward `Bearer ${session.access_token}`.

## 6. Rate limiting вЂ” FOUND ISSUE
- вќЊ No server-side usage/limit check in the edge function. The client checks `checkUsage` before invoking, but a direct call bypasses it в†’ unbounded Anthropic spend.
- **Planned fix (item 6):** after auth, read `usage.worksheets_used` for the month with the service role and return **429** if `>= FREE_LIMIT` (pro check stubbed `false` for now).
- вљ пёЏ **Important limitation:** because of the #2 caveat (users can write their own `usage` row), a determined user could reset `worksheets_used` to dodge the 429. The robust fix is to make `usage` writes **service-role only** (remove the user UPDATE/INSERT policies; have the edge function increment). Recommend as a fast-follow вЂ” out of this session's "don't over-engineer" scope, but flagged.

## Fixes applied (Session 10)
- **#5 Edge auth + ownership** вЂ” `fill-worksheet` now validates the caller's JWT
  via the **userClient pattern** (anon client forwarding the `Authorization`
  header в†’ `getUser()`), then verifies `worksheet.user_id === user.id` (401/403).
  `fillWorksheet()` forwards `Bearer ${session.access_token}`.
  **Verified live:** anon key в†’ `401 Unauthorized`; deployed.
- **#6 Rate limit** вЂ” server-side monthly `usage` check returns **429** at the
  free limit, and counts each attempt server-side (service role) *before* the
  Anthropic call, capping spend at `FREE_LIMIT`/month. *(Trade-off: a failed
  worksheet still consumes a credit вЂ” the secure default; move to on-success
  later if you prefer.)*
- **Usage hardening** вЂ” removed the client write policies on `usage` (see
  `supabase/migrations/20260629_security_hardening.sql`); client `incrementUsage`
  deleted. Users can no longer reset their counter; only the service role writes it.
- **#3 Storage** вЂ” policies (re)created in the same migration file вЂ” **run it**.
- **.gitignore** вЂ” broadened to `.env`, `.env.*`, with `!.env.example`.
- **CORS** вЂ” kept `'*'` with a comment explaining it's correct for a mobile-only
  app (RN sends no Origin); flagged to allowlist if a web client is ever added.
