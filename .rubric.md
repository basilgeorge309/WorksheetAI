Six pass/fail checks (2 per feature).

## Part 1 — Terms of Service
- [ ] **1. `docs/terms.html` exists and matches privacy.html styling.** Contains all
      required sections (Acceptance, Description, User Responsibilities incl.
      "homework/practice only, not graded exams", Subscription Terms 3/mo +
      $4.99/$29.99 auto-renew, IP, Disclaimer, Limitation of Liability,
      Termination, Changes, Contact support@scribbl.app, Last updated June 29,
      2026). White bg / #1A1A1A text / #2563EB accent / max-width 680px / mobile.
- [ ] **2. Settings LEGAL has a Terms row above Privacy Policy.** Opens
      `…/WorksheetAI/terms.html` via `Linking.openURL`, same row styling
      (TouchableOpacity 0.7, 15/#1A1A1A, chevron-forward, pv14, 1px border) as the
      existing Privacy/Support rows.

## Part 2 — Push Notifications (local only)
- [ ] **3. `lib/notifications.ts` + config wired, Expo-Go-safe.** Exports
      `registerForPushNotifications` (Device guard, permission flow, Android
      channel), `savePushToken` (writes `profiles.push_token`), and
      `sendLocalNotification`. Native calls are guarded so Expo Go can't crash.
      `app.json` adds the expo-notifications plugin (icon + #2563EB); migration
      `20260629_push_tokens.sql` adds `profiles.push_token`.
- [ ] **4. Registered on login + fires on completion.** `_layout` calls
      `registerForPushNotifications()` → `savePushToken` when a user is present.
      `index.tsx` sends the "Worksheet ready" local notification right after a
      successful `fillWorksheet()` and before navigation.

## Part 3 — In-App Account Deletion
- [ ] **5. `DeleteAccountModal` + Settings entry.** Bottom-sheet modal requires
      typing `DELETE` (button disabled until input === "DELETE"), uses a #DC2626
      delete button with a loading state and a Cancel text button; Settings shows
      a "Delete Account" row (#DC2626, pv14) below Sign out that opens it. No
      native `Alert`.
- [ ] **6. `deleteAccount()` + `delete-account` edge function.** `lib/auth.ts`
      `deleteAccount()` invokes the function with the auth token then signs out.
      The edge function validates the JWT (userClient), deletes the user's storage
      files, calls `auth.admin.deleteUser` (cascades to profiles/worksheets/usage),
      returns `{success}`. Deployed; unauth call → 401.
