# Scribbl — Session 7 Rubric (Image Upload + Camera Support)

Five pass/fail checks. Add camera + photo-library worksheet input alongside PDF.

## Checks

- [ ] **1. Three source chips render + select correctly.** Home screen shows a
      row of three equal-width 48px chips — PDF (document-outline), Camera
      (camera-outline), Photos (image-outline) — selected chip styled
      border #2563EB / bg #EFF6FF, unselected #E5E5E5 / #FFFFFF. After a
      selection the preview card shows: PDF → filename + "Change"; image → a
      60x60 rounded (border 1px #E5E5E5) thumbnail + filename + "Change". No
      inline styles (StyleSheet.create only).

- [ ] **2. Camera + Photos handlers work and fail gracefully.** Camera and
      Photos each request permission via the installed expo-image-picker /
      expo-camera API; a denied permission shows an inline #DC2626 message
      (never `Alert()`). On success the handlers set
      `fileType='image'`, `fileUri`, `fileName`, `thumbnail`; cancel is a no-op.
      The existing PDF (document-picker) path still works and sets
      `fileType='pdf'`.

- [ ] **3. `uploadWorksheet` handles both types.** Signature is
      `(uri, userId, fileType: 'pdf' | 'image')`. It builds the path/ext/mime
      correctly (`.pdf`+application/pdf vs `.jpg`+image/jpeg), validates size
      (10MB PDF / 20MB image) before upload with an inline error, uploads bytes
      with the right `contentType`, inserts the `pending` worksheet row, and
      returns `{worksheetId, storagePath}` or `{error}`. `handleFill` passes the
      selected `fileType`.

- [ ] **4. Edge function branches on file type.** It reads the extension from
      `storagePath`; PDFs → `type:'document'` (application/pdf), images
      (jpg/jpeg/png/webp) → `type:'image'` with the correct `media_type`. The
      content block is passed as `[contentBlock, {type:'text', text:prompt}]`.
      The prompt text and positioning logic are unchanged. Function redeploys and
      the empty-body smoke test still returns the structured 400.

- [ ] **5. Compiles, permissioned, scoped.** `tsc --noEmit` passes (strict); the
      iOS bundle exports with zero resolution errors. `app.json` adds the
      expo-camera (cameraPermission) and expo-image-picker (photosPermission)
      plugin config. `ANTHROPIC_API_KEY` stays server-side only. No changes to
      auth, RevenueCat, or the history screen. Storage policy is confirmed (the
      existing `uploads/{uid}/*` wildcard already permits images — SQL note
      printed).

## Explicitly out of scope this session
- No video, no multi-page image, no crop/edit UI
- No edge-function prompt/positioning changes
- No auth / RevenueCat / history changes
