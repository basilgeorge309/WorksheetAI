# WorksheetAI — Session 3 Rubric (Supabase Auth)

Five pass/fail checks that define "done" for this session. Auth wiring + auth
gate only — no usage tracking, no RevenueCat, no PDF flow.

## Checks

- [ ] **1. Real Supabase client + auth helpers exist and compile.**
      `lib/supabase.ts` is the real `createClient` (AsyncStorage storage,
      autoRefreshToken, persistSession, detectSessionInUrl:false). `lib/auth.ts`
      exports `signInWithGoogle`, `signInWithApple`, `signInWithEmail`,
      `signUpWithEmail`, `signOut`, `getCurrentUser` — each returns
      `{ data, error }` and never throws (all wrapped in try/catch).
      `tsc --noEmit` passes and the iOS bundle exports with zero resolution errors.

- [ ] **2. AuthContext provides global auth state.** `context/AuthContext.tsx`
      exports `AuthProvider` + a `useAuth()` hook exposing `{ user, session,
      loading, signOut }`. It subscribes to `supabase.auth.onAuthStateChange`,
      starts `loading:true`, and flips to `false` after the first auth state
      resolves. The subscription is cleaned up on unmount.

- [ ] **3. App is wrapped + auth-gated in `app/_layout.tsx`.** Whole tree is
      inside `<AuthProvider>`. Gate: no session and not on `/onboarding` →
      redirect to `/onboarding`; session present and on `/onboarding` →
      redirect to `/(tabs)/`. Routing waits until `loading` is false (no
      redirect thrash before auth resolves).

- [ ] **4. Step 5 has the three auth options, inline errors, loading states.**
      `StepFreetier` renders: Apple button (black #000, white text), Google
      button (white + #E5E5E5 border, dark text) — both 52px / radius 8 /
      size 16 / weight 600 — an "── or ──" hairline divider, Email + Password
      `TextInput`s (password `secureTextEntry`), a Sign up/Sign in
      `OnboardingButton` whose label toggles on `isSignUp`, and a
      "Already have an account? Sign in" toggle (#2563EB, size 13). Errors render
      inline in #DC2626 (size 13) — never `Alert()`. While a request is in
      flight all buttons disable and the tapped button shows an
      `ActivityIndicator`. On success: set AsyncStorage `onboarding_complete`
      = `'true'` then navigate to `/(tabs)/`. No new colors beyond the palette.

- [ ] **5. Sign out works from Settings.** `settings.tsx` has a "Sign out"
      button (#DC2626) that calls `signOut()` from `useAuth()`; on success it
      clears AsyncStorage `onboarding_complete` and navigates to `/onboarding`.
      Email/password path is testable in Expo Go; native Apple/Google paths are
      lazy-loaded so they don't crash Expo Go at import (documented as
      dev-build-only).
