# WorksheetAI — Session 4 Rubric (PDF Upload + Anthropic Edge Function)

Six pass/fail checks. This is the first real end-to-end loop:
pick PDF → upload → edge function (Anthropic + pdf-lib) → download/preview.

## Checks

- [ ] **1. Edge function is spec-complete and fail-safe.**
      `supabase/functions/fill-worksheet/index.ts` accepts the POST body
      (`worksheetId, storagePath, style, difficulty, subject`), downloads the PDF
      from Storage with the **service role key**, extracts text, calls Anthropic
      **`claude-sonnet-4-6`** with `max_tokens: 1000` and the exact system prompt,
      strips markdown fences before `JSON.parse`, writes answers with pdf-lib
      (Helvetica, size 11, `rgb(0.1,0.1,0.5)`, top=700/middle=400/bottom=150,
      left margin 72, 40-unit vertical spacing), uploads to
      `outputs/{worksheetId}.pdf`, sets the worksheet row to `complete` +
      `output_path`, and returns `{success:true,outputPath}`. Every failure path
      sets status `error` and returns `{success:false,error}` — the function
      **always** returns a response. It logs Anthropic input/output tokens +
      latency.

- [ ] **2. `lib/worksheet.ts` — four helpers, correct contracts.**
      `uploadWorksheet` rejects >10MB before upload, reads bytes via
      expo-file-system, uploads to `uploads/{userId}/{timestamp}.pdf`, inserts a
      `worksheets` row with status `pending`, returns `{worksheetId, storagePath}`
      or `{error}`. `fillWorksheet` invokes the function then polls the row every
      2s until status is neither `pending` nor `processing` (max 30s) and returns
      `{outputPath}` or `{error}`. `checkUsage` reads `usage` for the current
      `YYYY-MM` month (limit 3) → `{used, limit, canUse}`. `incrementUsage`
      upserts/increments `worksheets_used` for the month. Every helper surfaces
      `{error}`, never throws.

- [ ] **3. Home screen runs the real flow with correct gating.** State matches
      spec; `checkUsage` runs on mount and on focus (`useFocusEffect`). Upload
      card shows empty vs selected-file states; style + difficulty chip rows;
      "Fill it in →" disabled when no file / loading / no usage. On tap the order
      is: **usage gate → upload → incrementUsage → fillWorksheet → navigate** to
      `/worksheet/{id}?outputPath=…`. >10MB and 0-usage show inline errors
      (#DC2626, no `Alert`). The app **never navigates while `loading` is true**
      (navigation only after `setLoading(false)`).

- [ ] **4. Worksheet output screen previews + exports.** Reads `worksheetId` +
      `outputPath` params, renders the PDF via WebView from a **signed** Storage
      URL (1h) with an `ActivityIndicator` while loading, and has Download
      (expo-file-system) + Share (expo-sharing) buttons each with their own
      loading state, plus a back button. Errors surface inline, no crash.

- [ ] **5. History screen lists real worksheets.** Queries the current user's
      `worksheets` (newest first, limit 20); each row shows filename, formatted
      date ("Jun 29"), and a status badge colored complete=#16A34A /
      processing=#2563EB / error=#DC2626; tapping a complete row navigates to the
      worksheet screen. Empty state and animated skeleton loading rows (opacity
      0.3↔0.7) are present.

- [ ] **6. Compiles, safe, and deployable.** `tsc --noEmit` passes (strict) and
      the iOS bundle exports with zero resolution errors. `ANTHROPIC_API_KEY`
      appears in **no** client file (only the edge function). Zero inline styles
      in touched RN screens. The session prints: SQL for the `worksheets` +
      `usage` tables with RLS, the Storage bucket policy SQL, and the exact
      `supabase functions deploy` + `secrets set` commands.
