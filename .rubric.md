# Scribbl — Session 17 Rubric (Edge Case Hardening)

15 checks. Goal: no edge case leaves a frozen spinner or unexplained white screen.
No design/styling changes; no AI-prompt/positioning changes.

## Part 1 — Network
- [ ] **1. Upload timeout (1.1).** `uploadWorksheet` races the Storage upload
      against a 30s timeout (`Promise.race`); on timeout returns
      "Upload timed out. Check your connection and try again." Never hangs.
- [ ] **2. Poll network failure + ceiling (1.2).** `fillWorksheet`'s poll keeps
      to the 30s ceiling even on network errors (no infinite retry); a poll
      network failure returns "Lost connection while processing. Check History to
      see if it finished." (not a raw error / not a status-failure message).
- [ ] **3. Connectivity pre-check (1.3).** `@react-native-community/netinfo`
      installed; `lib/network.ts` exports `isConnected()`; Home's fill flow checks
      it first and shows "No internet connection. Connect and try again." when
      offline.

## Part 2 — Bad files
- [ ] **4. Empty answers → 422 (2.1).** Edge function: `answers.length === 0`
      sets the row `status: 'error'` + stores the message and returns 422 with
      "Couldn't find any questions on this worksheet. Try a clearer photo or a
      different file." Redeployed.
- [ ] **5. >30 question cap (2.2).** Edge function renders only the first 30
      answers (stores full count in `answer_count`); celebration subtitle shows
      "{30} of {total} questions answered" when total > 30. No crash/silent drop.
- [ ] **6. Size boundary (2.3).** Confirmed `uploadWorksheet` uses `>` (exactly
      10MB PDF / 20MB image accepted; +1 byte rejected) with the existing message.
- [ ] **7. Duplicate-tap lock (2.4).** A synchronous guard (ref/`isSubmitting`)
      makes "Fill it in" truly single-flight — a rapid double-tap creates only one
      worksheet row; the guard releases on every exit path.

## Part 3 — Auth
- [ ] **8. Re-signup behavior documented (3.1).** Comment in `lib/auth.ts`
      `deleteAccount` notes `admin.deleteUser` frees the email for immediate
      re-signup (no code change).
- [ ] **9. Session expiry (3.2).** `fillWorksheet` checks `getSession()` first and
      returns "Your session expired. Please sign in again." if missing/expired;
      Home catches that specific error and routes to re-auth (sign out →
      onboarding) rather than inline text.
- [ ] **10. OAuth cancel silent (3.3).** Confirmed `signInWithGoogle` returns
      `{ message: 'cancelled' }` on cancel/dismiss and the UI stays silent.

## Part 4 — Payments
- [ ] **11. Sub-check fail-open (4.1).** `checkUsage` wraps `isProUser()` in
      try/catch; a RevenueCat failure logs + falls through to the DB free-tier
      check (never blocks a paying user).
- [ ] **12. External cancel detected (4.2).** Comment documents that
      `isProUser()` → `getCustomerInfo()` reflects App/Play Store cancellations
      (queried fresh, not a stale local flag).
- [ ] **13. Restore failure surfaced (4.3).** `restorePurchases` returns
      `{ success, error? }` with a clear failure message; `PaywallModal` and
      `settings.tsx` display it inline on failure.
